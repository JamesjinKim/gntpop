# GnT POP 비전: Smart Factory 2030

> **"데이터 기반 자율 제조 혁신"**
> DX에서 AX로, FOMs CPS를 통한 스마트팩토리 실현

---

## 1. 비전 개요

### 1.1 배경

2026년, 제조업은 **디지털 전환(DX)**을 넘어 **AI 전환(AX)** 시대로 진입하고 있습니다.

- **DX (Digital Transformation)**: 연결과 데이터화 - "모든 업무가 데이터로 기록되는가?"
- **AX (AI Transformation)**: 판단과 최적화 - "AI가 인간 대신 판단·실행하는가?"

> DX(도로)가 없으면 AX(자율주행차)는 달릴 수 없습니다.
> 반대로 AX 없는 DX는 차 없는 텅 빈 고속도로와 같습니다.
> **두 전환이 함께 맞물릴 때, 비로소 진정한 기업 혁신이 완성됩니다.**

### 1.2 비전 선언

```

│                                                                 │

│    AI 기반 자율 운영 스마트팩토리를 실현한다"                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. DX와 AX의 결정적 차이

### 2.1 핵심 비교

| 구분 | DX (Digital Transformation) | AX (AI Transformation) |
|------|----------------------------|------------------------|
| **정의** | 디지털 기술로 기업 체질 혁신 | AI를 핵심 동력으로 지능화 |
| **핵심 키워드** | 연결 · 데이터화 | 판단 · 최적화 |
| **주요 질문** | "모든 업무가 데이터로 기록되는가?" | "AI가 인간 대신 판단·실행하는가?" |
| **비유** | 잘 닦인 고속도로와 신호등 | 그 위를 달리는 자율주행 시스템 |
| **목표** | 업무 효율화, 데이터 축적 | 의사결정 자동화, 자율 운영 |

### 2.2 DX 전환을 위한 3대 핵심 요소

| 요소 | 역할 | GnT 적용 |
|------|------|---------|
| **클라우드 (Cloud)** | 데이터의 심장, 언제 어디서나 정보 접근 | AWS/Azure 기반 POP 클라우드화 |
| **빅데이터 & 분석** | 원석 데이터에서 인사이트 도출 | 4M 데이터 분석, 생산성 지표 |
| **AI & 머신러닝** | 분석된 데이터를 학습, 판단·진화 | 품질 예측, 설비 예지정비 |

### 2.3 AX가 만들어내는 변화

| 영역 | 현재 (수작업) | DX 전환 | AX 전환 |
|------|-------------|---------|---------|
| **생산실적** | 수기 기록 | 바코드 스캔 자동입력 | AI가 이상치 자동 감지 |
| **품질검사** | 작업자 육안 | 디지털 측정/기록 | AI 이미지 분석 자동판정 |
| **설비관리** | 고장 후 대응 | 실시간 상태 모니터링 | AI 예지정비 (고장 예측) |
| **생산계획** | 경험 기반 수립 | 데이터 기반 수립 | AI 자동 최적화 스케줄링 |
| **고객 응대** | 담당자 직접 응대 | 시스템 자동 알림 | AI 챗봇 24시간 자동 응답 |

## 3. FOMs CPS 프레임워크

### 3.1 FOMs Package 개념

**FOMs (Factory Operation Management System)**는 정보기술(IT), 자동화기술(AT), 운영기술(OT)가 융합된 **융합기술(CT)**입니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    FOMs Package (CT)                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐        │
│   │     IT      │  │     AT      │  │     OT      │        │
│   │ Information │  │ Automation  │  │ Operation   │        │
│   │ Technology  │  │ Technology  │  │ Technology  │        │
│   ├─────────────┤  ├─────────────┤  ├─────────────┤        │
│   │ • ERP/MES   │  │ • PLC       │  │ • 공정 관리   │        │
│   │ • 데이터 관리  │  │ • 로봇       │  │ • 현장 운영   │        │
│   │ • 클라우드    │  │ • 설비 제어   │  │ • 품질 관리    │        │
│   │ • BigData   │  │ • IoT 센서   │  │ • 작업 지시    │        │
│   └─────────────┘  └─────────────┘  └─────────────┘        │
│                           ↓                                │
│              ┌─────────────────────────┐                   │
│              │   CT (Convergence)      │                   │
│              │   융합기술                │                   │
│              │   IT + AT + OT 통합      │                   │
│              └─────────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 FOMs Cycle 구성요소

| 단계  | 구성요소  | 설명  | GnT 적용  |
|------|---------|------|---------|
| 1 | **FOM-Logic** | 4M 데이터 전처리 | Man, Machine, Material, Method 데이터 수집 |
| 2 | **FOM System Analysis** | 정보화 분석/컨설팅 | 생산성 분석, 병목 공정 식별 |
| 3 | **CPS Analysis** | 3D 시뮬레이션 가상공장 | Digital Twin 구축, 물류 최적화 |
| 4 | **디지로그 융합** | SW 기반 맞춤형 교육 | 작업자 역량 강화, 스마트 운영 |

### 3.3 CPS (Cyber-Physical System) 개념

**CPS(사이버물리시스템)**는 스마트팩토리의 궁극적 지향점입니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    CPS Architecture                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Physical World (현실 세계)     Cyber World (가상 세계)        │
│   ┌─────────────────────┐       ┌─────────────────────┐    │
│   │                     │       │                     │    │
│   │   제조 현장           │◄─────►│   Digital Twin      │    │
│   │   • 설비             │ 실시간 │   • 가상 공장          │    │
│   │   • 작업자           │ 동기화 │   • 시뮬레이션          │    │
│   │   • 자재             │       │   • 예측 모델         │    │
│   │   • 제품             │       │   • 최적화 엔진        │    │
│   │                     │       │                     │    │
│   └─────────────────────┘       └─────────────────────┘    │
│              │                            │                 │
│              ▼                            ▼                 │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              인지 → 판단 → 행동                        │  │
│   │         (Sensor) (Decision) (Actuation)             │  │
│   │                  자율 운영                            │  │
│   └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 5. GnT POP 단계별 발전 계획

### 5.1 Level 1: DX 기반 (2026 상반기) - 현재

**목표**: 제조 현장의 디지털화, 실시간 데이터 수집

| 영역    | 기능                        | 상태   |
|--------|----------------------------|-------|
| 생산관리 | 작업지시, 생산실적 입력, LOT 추적 | 개발중  |
| 품질관리 | 검사결과 입력, 불량 기록         | 개발중  |
| 설비관리 | 설비 상태 모니터링              | 개발중  |
| 대시보드 | KPI 현황, 실시간 모니터링        | ✅ 완료 |

**핵심 질문**: "모든 생산 데이터가 실시간으로 기록되는가?"

### 5.2 Level 2: FOMs 적용 (2026 하반기)

**목표**: IT-AT-OT 융합, 4M 데이터 분석 체계 구축

| 영역 | 기능 | 기술 |
|------|------|------|
| FOM-Logic | 4M 데이터 전처리 | ETL 파이프라인, 데이터 정규화 |
| IT 연동 | ERP/MES 연계 | API 연동, 데이터 동기화 |
| AT 연동 | 설비 데이터 수집 | PLC 통신, IoT 센서 |
| OT 연동 | 현장 운영 최적화 | 공정 분석, 병목 식별 |

**핵심 질문**: "IT-AT-OT가 유기적으로 연결되어 분석 가능한가?"

### 5.3 Level 3: CPS 구축 (2027)

**목표**: 사이버물리시스템 구축, Digital Twin 실현

| 영역 | 기능 | 기술 |
|------|------|------|
| Digital Twin | 가상 공장 모델링 | 3D 시뮬레이션 |
| 실시간 동기화 | 현실-가상 데이터 연동 | 실시간 스트리밍 |
| 시뮬레이션 | 생산 계획 검증 | What-if 분석 |
| 물류 최적화 | 자재 흐름 최적화 | 경로 최적화 알고리즘 |

**핵심 질문**: "가상 공장에서 미리 검증하고 최적화할 수 있는가?"

### 5.4 Level 4: AX 전환 (2028+)

**목표**: AI 기반 자율 운영 스마트팩토리

| 영역 | 기능 | AI 기술 |
|------|------|---------|
| 품질 예측 | 불량 사전 감지 | ML 이상 탐지 |
| 예지정비 | 설비 고장 예측 | 시계열 예측 모델 |
| 생산 최적화 | 자동 스케줄링 | 강화학습, 최적화 |
| 자율 운영 | 자동 의사결정 | LLM 기반 에이전트 |

**핵심 질문**: "AI가 스스로 판단하고 최적의 결정을 실행하는가?"

## 6. 기대 효과 및 KPI

### 6.1 단계별 기대 효과

| Level | 기대 효과 | 목표 지표 |
|-------|----------|----------|
| **Level 1** (DX) | 수작업 제거, 데이터 축적 | 데이터 수집률 95% |
| **Level 2** (FOMs) | 분석 기반 의사결정 | 생산성 10% 향상 |
| **Level 3** (CPS) | 사전 검증, 리스크 감소 | 불량률 30% 감소 |
| **Level 4** (AX) | 자율 운영, 최적화 | 설비 가동률 95% |

## 7. 조직 문화 혁신

### 7.1 데이터 중심 문화

> **"기술보다 더 중요한 것은 '데이터로 사고하는 조직 문화'입니다."**

| 기존 문화 | 데이터 중심 문화 |
|----------|-----------------|
| 경험 기반 의사결정 | 데이터 기반 의사결정 |
| 문제 발생 후 대응 | 예측 기반 사전 대응 |
| 개인 역량 의존 | 시스템 기반 표준화 |
| 부서별 사일로 | 데이터 통합/공유 |

### 7.2 변화 관리 전략

1. **인식 전환**: DX/AX 필요성 교육, 성공 사례 공유
2. **역량 강화**: 데이터 리터러시 교육, AI 활용 훈련
3. **참여 유도**: 현장 의견 수렴, 개선 아이디어 보상
