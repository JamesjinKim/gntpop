## 1. 핵심 아이디어

신호테크놀로지의 **WDAQ 엣지 AI 이상 조기탐지 기술**(삼성디스플레이 납품)을
중소기업에 맞게 간소화하고, **Mini MES 생산관리 SaaS**와 결합하여
중소 제조업체를 위한 **품질 관리 + 생산성 향상 올인원 솔루션**을 만든다.

> **"센서 박스 하나로, 품질 사고 예방부터 생산 관리까지 한번에"**

### 키워드 방향 (필수 준수)

| ✅ 사용 | ❌ 최소화 |
|---------|----------|
| 품질 사고 예방 | ~~예지보전~~ |
| 공정 품질 관리 | ~~고장 예측~~ |
| 이상 조기 감지 | ~~설비 수명 연장~~ |
| 충돌 예방 | ~~Predictive Maintenance~~ |
| 생산성 향상 | |
| 불량률 감소 | |

> **현장 실정:** 디스플레이 장비는 오버스펙 설계로 고장 빈도가 낮다.
> 고장 예측보다 **충돌/사고 예방**, **공정 품질 이탈 사전 감지**가 진짜 수요.
> 이는 중소 제조업에서도 동일 — 설비가 멈추는 것보다 **불량이 나오는 것**이 더 큰 손해.

## 2. 정부지원사업 기반 기술 (신호테크놀로지)

### 과제 개요

| 항목 | 내용 |
|------|------|
| **사업명** | 2026 지역자율형 디지털혁신프로젝트 — SW융합 디스플레이 장비개발 고도화 |
| **과제명** | 디스플레이 공정의 Dark Factory 지향 실시간 이상 조기탐지를 위한 오감센서 기반 엣지 AI 시스템 개발 |
| **주관기관** | (주)신호테크놀로지 |
| **수요기업** | 삼성디스플레이 (탕정 A2/A3/A4 라인, 400+ 장비) |
| **개발 제품** | WDAQ — Edge AI 기반 다중센서 지능형 상태감시 시스템 |
| **총 사업비** | 108백만원 (정부 90백만 + 민간 18백만) |
| **수행기간** | 2026.05 ~ 2026.11 (7개월, 3 iteration 진화적 개발) |
| **매출 목표** | 580백만(2026) → 1,250백만(2027) → 2,350백만(2028) |

### 4계층 아키텍처

```
L1: 센서 계층 — 10종 오감센서 (진동/온도/습도/압력/전류/전압/오존/조도/열화상/카메라)
    ↓ MODBUS/RS485, I2C, UART, SPI        ※ L1은 데이터 수집만, 추론 없음
L2: 엣지 AI 계층 — WDAQ (RPI 8GB + Hailo-8L NPU 13 TOPS)
    │ ├─ 단계1: 룰 기반 임계치 판정 (100ms 이내) ← AI 아님, 룰 기반
    │ └─ 단계2: Autoencoder 복합 분석 (1.5초 이내) ← AI는 여기서만
    ↓ MQTT / UDP
L3: 게이트웨이 계층 — FastAPI + InfluxDB + SQLite(72h 롤링) + CSV(장기 보관)
    ↓ REST API / WebSocket
L4: 대시보드 계층 — React 웹앱 (실시간 모니터링/의사결정 지원)
```

### 핵심 기술 차별점: 인간-AI 협업 형식지화 사이클

숙련자의 암묵지(소리, 진동 감지 등)가 퇴직과 함께 소실되는 문제를 해결:

```
① 룰 기반 임계치 (L2 단계1)
   엔지니어의 도메인 지식을 명시적 규칙으로 코딩 → 1차 형식지화
        ↓
② Autoencoder (L2 단계2)
   숙련자가 "정상"이라고 판단한 데이터의 통계적 패턴 학습
   → 알려지지 않은 이상(Unknown Unknown) 감지
        ↓
③ XAI 기여도 분해
   어느 센서가 이상에 기여했는지 단서 제공
   → 엔지니어가 해석 후 룰 정교화 → ①로 환류
```

> **⚠️ 올바른 표현:** "숙련자가 정상이라 판단한 데이터 기반 AI 이상탐지"
> **❌ 금지 표현:** "AI가 암묵지를 직접 학습한다"

### 6대 성과지표

| No | 지표 | 목표값 |
|----|------|--------|
| 1 | AI 조기감지 정확도 (F1-Score) | 90% 이상 |
| 2 | 복합센서 분석 속도 | 1.5초 이내 |
| 3 | 단일센서 판정 시간 | 100ms 이내 |

| 6 | 네트워크 단절 자율운영 | 72시간 이상 |

### 6대 핵심 키워드

| No | 키워드 | 정의 |
|----|--------|------|
| 1 | **AIoT 이상탐지** | AI+IoT 통합 지능형 이상 감지 |
| 2 | **오감센서** | 10종 다중센서로 공정 모든 정보 수집 |
| 3 | **품질데이터 수집** | 실시간 센서 데이터 수집·저장 → AI 분석 기반 |

   │                 │          │                 │
   │ • 생산 카운트    │          │ • 진동/온도/습도  │
   │ • 양품/불량 수   │          │ • 전류/전압      │
   │ • 사이클 타임    │          │ • 압력/오존/조도  │
   │ • 가동/비가동    │          │ • 열화상/카메라   │

            │                             │
            └──────────┬──────────────────┘

                       │
         ┌─────────────┴─────────────┐
         ▼                            ▼
┌─────────────────┐        ┌─────────────────┐

### POP vs WDAQ: 역할 분리 (핵심 구분)

| 구분 | POP (Point of Production) | WDAQ (Edge AI 센서) |
|------|--------------------------|---------------------|
| **데이터 출처** | 생산장비 자체 (PLC/컨트롤러) | 외부 부착 센서 (신호테크 하드웨어) |
| **수집 정보** | 생산 카운트, 양품/불량 수, 사이클 타임, 가동/비가동 | 진동, 온도, 습도, 전류, 압력 등 장비 상태 |
| **역할** | "얼마나 만들었나, 불량이 몇 개인가" | "불량이 나올 때 장비 상태는 어떠했나" |
| **성격** | 생산 결과 데이터 | 장비 환경/컨디션 데이터 |
| **연동 방식** | 장비 PLC → 프로토콜 통신 (OPC-UA, MODBUS 등) | WDAQ 센서 → MQTT/UDP → 게이트웨이 |

> **핵심 가치: POP + WDAQ를 결합할 때 진짜 힘이 나온다**
>
> - POP만: "불량 10개 나왔다" (결과만 알 수 있음)
> - WDAQ만: "진동이 올라갔다" (원인 후보만 있음)
> - **POP + WDAQ**: "불량 10개 나왔을 때 진동 125Hz, 온도 78°C였다" → **품질 원인 추적 가능**
>
> 이것이 이카운트, vSmart 모두 제공하지 못하는 Factory Box만의 차별점

### 왜 2트랙인가

| 문제 | 1트랙만 (올인원) | 1트랙만 (연동형) | **2트랙 (병행)** |
|------|----------------|----------------|-----------------|
| MES만으론 ERP 없이 불편 | ❌ 약점 노출 | 해당 없음 | ✅ Track2로 우회 |
| 기도입 공장 포화 우려 | 해당 없음 | ❌ 시장 제한적 | ✅ Track1로 미도입 공략 |
| 이카운트와 가격 경쟁 | ❌ 직접 경쟁 | ✅ 보완재 | ✅ 경쟁 회피 |
| 시장 커버리지 | 미도입만 | 기도입만 | ✅ **양쪽 모두** |

### Track 1: Mini MES + WDAQ 올인원

| 항목 | 내용 |
|------|------|
| **타겟** | ERP/MES 없는 소규모 공장 (10~50인), 131,000개 시장 |
| **제안** | "아무것도 없어도 됩니다. 이것 하나면 생산관리 + 품질감시 시작" |
| **구성** | WDAQ HW + Mini MES SaaS (생산실적, 작업지시, 대시보드, 현황판, 품질감시) |
| **가격** | HW 200~300만 + 월 15~20만 |
| **강점** | 올인원이라 도입 간단, 장벽 최소 |
| **한계** | 회계/인사/재고 없음 → 성장 시 ERP 별도 필요 |
| **대응** | 이카운트(월 4만) 병행 안내, 향후 Track 2 전환 가능 |

### Track 2: WDAQ + 기존 ERP/MES 연동

| 항목 | 내용 |
|------|------|
| **타겟** | 이카운트/기타 ERP/MES 이미 쓰는 공장, 32,000개 시장 (특히 기초수준 24,000개) |
| **제안** | "지금 쓰시는 ERP는 그대로. AI 품질 눈만 얹어드립니다" |
| **구성** | WDAQ HW + 품질감시 대시보드 + 기존 시스템 데이터 연동 + 현황판 |
| **가격** | HW 300~500만 + 월 20~30만 (연동 커스텀 포함) |
| **강점** | 기존 투자 보호, 고도화 명분 → 정부 보조금 활용 용이 |
| **한계** | 연동 개발 공수, ERP마다 다름 |
| **대응** | 이카운트 API 연동 1순위 개발 (시장 점유율 최대) |

### MES 표준 공통 모듈 전략 (업종별 템플릿 ❌)

**원칙: 표준 공통 모듈만 먼저 개발, 업종별 커스텀은 영업 상황에 따라 개발**

여러 업종(사출/프레스/CNC 등)의 템플릿을 사전에 준비하는 것은 비효율적.
실제 고객이 확보되었을 때 해당 업종에 맞게 커스터마이징하는 것이 현실적.

#### MES 표준 공통 모듈 (업종 무관, 선개발)

| 모듈 | 기능 | 비고 |
|------|------|------|
| **작업지시** | 생산 오더 등록/조회/수정, 공정 흐름 설정 | 모든 제조업 공통 |
| **생산실적** | 모바일 2~3터치 수량/불량 입력, POP 연동 | 모든 제조업 공통 |
| **품질기록** | 검사 항목 등록, 불량 유형 분류, 이력 관리 | 모든 제조업 공통 |
| **대시보드** | 일별/주별 생산량, 불량률, 달성률 시각화 | 모든 제조업 공통 |
| **현황판** | 대형 모니터 전용 실시간 공장 상태 표시 | 모든 제조업 공통 |

#### 업종별 커스텀 (영업 확보 후 개발)

| 업종 | 커스텀 포인트 | 개발 시점 |
|------|-------------|----------|
| 사출 | 금형 관리, 사이클 타임 변동, 수지 온도/압력 | 1호 사출 고객 확보 시 |
| 프레스 | 타발 횟수, 금형 수명, 소재 투입 관리 | 1호 프레스 고객 확보 시 |
| CNC | 가공 프로그램 관리, 공구 수명, 칩 관리 | 1호 CNC 고객 확보 시 |
| 도금/표면처리 | 약품 농도, 온도, 침적 시간 | 1호 도금 고객 확보 시 |

> **투자 최소화**: 공통 모듈로 시작, 고객이 생길 때마다 업종 모듈 추가
> → 선투자 리스크 없음, 고객 니즈 기반 개발

#### 공통 자산 vs 트랙별 차이

| 구분 | 공통 (양 트랙) | Track 1 전용 | Track 2 전용 |
|------|-------------|-------------|-------------|
| WDAQ 하드웨어 | ✅ 동일 | - | - |
| AI 이상탐지 엔진 | ✅ 동일 | - | - |
| 품질 감시 대시보드 | ✅ 동일 | - | - |
| 공장 현황판 | ✅ 동일 | - | - |
| MES 표준 공통 모듈 | - | ✅ 포함 | ❌ 불필요 |
| POP 연동 프레임워크 | ✅ 공통 설계 | 직접 구현 | 기존 POP 연동 |
| ERP 연동 모듈 | - | ❌ 불필요 | ✅ 맞춤 개발 |

> **개발의 70~80%는 공통**, 트랙별 차이는 20~30%만 추가 개발

### 중소기업 4대 고통 (핵심 셀링 포인트 — 양 트랙 공통)

## 8. 신호테크놀로지 내 성장 전략

### 단계별 로드맵

| 단계 | 시기 | 행동 | 선생님의 입지 |
|------|------|------|-------------|
| **현재** | 진행 중 | 정부지원사업 수주 (WDAQ 대기업용 기술 확보) | R&D 이사, 과제 기획 주도 |
| **Phase 1** | ~6개월 | 정부과제 수행 + 중소기업용 간소화 버전 기획서 제안 | 신사업 라인 제안자 |
| **Phase 2** | 6~12개월 | Factory Box 제품화 + Mini MES SaaS 결합 | 사업부문 총괄 가능 |
| **Phase 3** | 1~2년 | 중소기업 고객 확보, 스마트공장 보급사업 공급기업 등록 | 매출 기여 + 연봉 협상력 |
| **Phase 4** | 2년~ | 제품 라인업 확장, 전국 확대, 서진메카트로닉스형 사례 복제 | 사업본부장 또는 독립 가능 |

### 핵심 논리: 대기업 기술 → 중소기업 제품화

```
삼성디스플레이 납품 (WDAQ)
    │
    │ 기술 검증 완료 + 레퍼런스 확보
    │
    ↓
중소기업용 간소화 (Factory Box)
    │
    │ + Mini MES SaaS 결합
    │ + 정부 보조금 활용
    │
    ↓
스마트공장 보급사업 공급기업 등록
    │
    │ 정부가 마케팅/영업 지원
    │
    ↓
중소 제조업체 다수 확보 → 월 구독 수입 누적
```

### 두 가지 경로 동시 추진

| 경로 | 내용 | 장점 | 고려사항 |
|------|------|------|---------|
| **A. 회사 내 신사업** | 신호테크 이름으로 Factory Box 제품화 | 회사 자원 활용, 안정적, WDAQ 기술 직접 활용 | 회사 의사결정 필요 |

**추천 전략**: A를 주력으로, B는 A에서 파생된 SaaS 부분만 독립 운영

- Factory Box HW(WDAQ) + AI → 신호테크 매출 (회사 성장 → 본인 입지)
